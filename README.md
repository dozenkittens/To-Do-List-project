# Техническое задание на разработку информационной системы
## «Планировщик задач (To-Do List)»

Версия 1.0

---

### 1. Общие сведения

**1.1. Наименование системы**
Полное наименование: Информационная система «Планировщик задач».
Краткое наименование: SmartTodo.

---

### 2. Назначение и цели создания системы

**2.1. Назначение системы**
Система предназначена для автоматизации процесса планирования личных задач и контроля их выполнения. Программный продукт позволяет пользователю создавать перечень задач, отслеживать их статус и получать наглядную статистику о прогрессе.

**2.2. Цели создания**
- Разработка удобного инструмента для повседневного планирования задач и повышения личной продуктивности пользователя
- Обеспечение быстрого доступа к списку задач с возможностью управления ими (добавление, выполнение, удаление)
- Визуализация прогресса выполнения задач для мотивации пользователя
- Получение практических навыков проектирования и разработки программных модулей студентом

---

### 3. Требования к программному продукту

#### 3.1. Требования к функциональным характеристикам

Система должна обеспечивать выполнение следующих функций:

**3.1.1. Управление задачами**
- Создание новой задачи. При создании пользователь должен указать название задачи (обязательное поле)
- Просмотр полного списка всех созданных задач
- Удаление задачи с предварительным подтверждением действия во избежание ошибочного удаления

**3.1.2. Учет выполнения задач**
- Отметка задачи как выполненной. Отметка должна проставляться одним кликом (нажатие на чекбокс)
- Возможность отмены отметки о выполнении (снятие чекбокса) в случае ошибочного действия
- Визуальное различие выполненных и активных задач (зачеркивание текста выполненной задачи)

**3.1.3. Статистика и аналитика**
- Подсчет и отображение общего количества задач в системе
- Подсчет и отображение количества выполненных задач
- Подсчет и отображение количества активных (невыполненных) задач

#### 3.2. Требования к пользовательскому интерфейсу
- Интерфейс должен быть одностраничным (Single Page Application), все элементы располагаются на одном экране
- Интерфейс должен быть интуитивно понятным и не требовать обучения пользователя
- Дизайн должен быть выполнен в минималистичном стиле с использованием плоских элементов (flat design)

#### 3.3. Требования к составу и параметрам технических средств
- **Клиентская часть (браузер):** любое устройство с доступом в интернет и установленным современным веб-браузером (Google Chrome, Яндекс.Браузер, Mozilla Firefox, Safari последних версий)

---

### 4. Состав и структура работ

Разработка системы осуществляется в несколько этапов:

- Проектирование структуры системы и написание технического задания
- Разработка UX/UI дизайна (создание структуры страницы из примитивов)
- Верстка пользовательского интерфейса (HTML, CSS)
- Программирование логики приложения (JavaScript)
- Тестирование и отладка
- Подготовка отчетной документации

---

### 5. Порядок контроля и приемки

Приемка системы осуществляется преподавателем по результатам выполнения всех этапов работ. Система считается принятой, если:

- [x] Весь заявленный функционал (раздел 3.1) работает без ошибок
- [x] Интерфейс соответствует утвержденному UX-дизайну (Приложение)
- [x] Код программы является читаемым и структурированным
- [x] Обеспечена корректная обработка пользовательских действий (добавление, удаление, отметка выполнения)

### 6. Содержание и структура пользовательского интерфейса

В связи с концепцией одностраничного приложения, система состоит из **одной главной страницы**, которая содержит все необходимые функциональные блоки.

#### 6.1. Детальное описание элементов страницы

**1. Шапка (Header)**
- **Левая часть:** логотип или название системы «SmartTodo»
- **Правая часть:** блок приветствия пользователя (например, «Привет, Пользователь!»)

**2. Дашборд (Блок статистики)**
- Три информационные карточки, расположенные горизонтально в ряд:
  - **Карточка «Всего задач»** — отображает общее количество задач (иконка 📋)
  - **Карточка «Выполнено»** — отображает количество выполненных задач (иконка ✅)
  - **Карточка «Активно»** — отображает количество невыполненных задач (иконка ⏳)
- Каждая карточка содержит иконку, название показателя и числовое значение

**3. Блок создания задачи**
- Текстовое поле для ввода названия новой задачи (placeholder: «Новая задача...»)
- Кнопка «+ Добавить», расположенная справа от поля ввода. Нажатие кнопки добавляет задачу в список

**4. Список задач**
- Область для отображения всех созданных задач в виде вертикального списка
- Каждый элемент списка (отдельная задача) содержит:
  - **Чекбокс** (квадратный элемент) слева для отметки выполнения
  - **Текст задачи** в центре элемента
  - **Иконка «Корзина»** (🗑️) справа для удаления задачи

**5. Визуальные состояния задач**

| Состояние | Чекбокс | Текст | Действие |
|:---------:|:-------:|:-----:|:--------:|
| Активная задача | [⬜] | Обычный (не зачеркнутый) | Можно отметить выполнение |
| Выполненная задача | [✅] | Зачеркнутый, серым цветом | Можно снять отметку |
